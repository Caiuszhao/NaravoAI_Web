- 产品名称：NaravoAI Website - Demo1

- 文档版本：v1.0
- 更新时间：2026-04-04
- 适用范围：`src/app/components/DemoFeed.tsx` 当前实现

## 2. 背景与目标
Demo1 是网站首页 Demo 区中的主交互样例。用户需要在短时间窗口内完成“快速点击破门”互动，系统根据点击结果进入不同结局分支视频。该模块不仅要展示交互叙事能力，还要在真实网络环境下保证过渡稳定、操作可理解、体验连贯。

本期目标：
- 用户能完整经历 `Intro -> Loop 互动 -> Branch 结局 -> Replay`。
- 交互规则直观，10 秒内可完成输入。
- 视频切换和加载过程尽可能平滑，减少黑屏和误闪。
- 页面内核心互动（暂停、恢复、评论、角色弹窗、点赞）行为可用。

## 3. 用户与使用场景
### 3.1 目标用户
- 首次访问官网、想快速理解产品交互能力的访客。
- 需要观看并体验“可分支叙事”的潜在客户。

### 3.2 核心场景
- 用户进入 Demo1，先看开场片段。
- 开场结束后自动进入互动片段（Loop）。
- 用户在倒计时内快速点击中间按钮。
- 系统根据点击次数进入不同结局。
- 结局结束后用户点击 Replay 重新开始。

## 6. 详细交互流程
### 6.1 初始进入
1. 进入 Demo1 时，先显示封面图和加载动画。
2. 顶部按钮、标题描述、底部操作栏保持可见。
3. 首帧可播后，封面层淡出，进入 Intro 播放。

### 6.2 Intro 阶段
- 自动播放 Intro。
- 允许用户点击视频区域暂停（点击 UI 元素不触发暂停）。
- 暂停后显示中间播放按钮，点击恢复。
- Intro 播放结束后自动切到 Loop。

### 6.3 Loop 互动阶段
- 自动进入全屏（若用户此前非全屏且未手动改动）。
- 展示交互按钮与倒计时环，顶部显示 Smash! 提示条。
- 倒计时：10 秒，与互动窗口一致。
- 用户点击中间按钮，记录点击次数并反馈按钮按压、波纹动效。
- 倒计时结束后，根据点击次数分支：
  - `>= 20` -> branch_click（Forced entry successful）
  - `10-19` -> branch_hold（Victory despite injury）
  - `< 10` -> branch_rapid（Forced entry failed）

### 6.4 分支切换阶段
- 在 Loop 开始后并行预加载三个分支视频。
- 分支跳转前要求目标分支至少达到 `loadeddata` 可播。
- 未就绪时显示 `Loading Branch` 过渡层。
- 分支可播后切换到对应结局视频。

### 6.5 分支结束与 Replay
- 分支播完后显示 `Replay` 按钮。
- 点击 Replay 必须从 Intro 第一段重新开始。
- 不允许出现“先短暂播放 Loop/分支再回到 Intro”的异常。

## 7. 页面元素与行为
### 7.1 顶部区域
- 左：Back 按钮
- 右：More 按钮
- 互动提示条在两侧按钮之间，保持安全边距，不重叠。

### 7.2 中部视频层
- 视频双槽 A/B 切换（预备槽加载后再激活）。
- 非活动 Demo 或离开当前屏幕时自动暂停。
- 返回当前屏幕后从当前位置继续播放（非手动暂停状态下自动恢复）。

### 7.3 底部信息与操作
- 标题（Demo1）
- 描述（两行截断，超出显示省略）
- 用户头像、Like、Comment、Cast、Fullscreen
- Like 点击变红，可再次取消。
- Comment 打开评论抽屉。
- Cast 打开角色抽屉。

## 8. 关键状态机（实现口径）
### 8.1 StoryPhase
- `intro`
- `loop`
- `branch_click`
- `branch_hold`
- `branch_rapid`

### 8.2 关键状态
- `isInitialFrameReady`：首帧是否就绪
- `isVideoPaused` / `isPausedByUser`：暂停状态
- `isBranchTransitionLoading`：分支切换加载中
- `showBranchReplay`：是否显示 Replay
- `activeVideoSlot` + `slotSources`：双视频槽控制

## 9. 性能与体验要求
- 首次进入优先加载 Intro，随后加载 Loop，再加载分支。
- 分支切换不应出现明显黑屏。
- 互动按钮点击反馈必须明显：缩放 + 波纹。
- 倒计时环从 12 点方向顺时针消失。
- 非当前屏 Demo 不播放，避免并行视频占用资源。

## 10. 异常与兜底策略
- 预加载失败不阻断整体流程，按可用资源继续播放。
- 自动播放受限时允许静音兜底，并在用户交互后恢复声音策略。
- 分支切换若事件丢失，需有 readyState 兜底检查，避免卡住。

## 11. 验收标准（QA Checklist）
### 11.1 播放与切换
- Intro 能正常播放并进入 Loop。
- Loop 阶段互动 UI 正常出现。
- 分支切换时无长时间黑屏。

### 11.2 互动规则
- 倒计时为 10 秒，和分支判定窗口一致。
- 点击次数在三个阈值区间进入对应分支。
- 按钮按压缩放、波纹反馈可见。

### 11.3 UI 与可用性
- 顶部提示条与左右按钮不重叠。
- 描述文本两行截断生效。
- 暂停按钮只在用户主动暂停时出现。
- 分支结束只出现 Replay，不与暂停按钮同时出现。

### 11.4 Feed 行为
- 滚动离开 Demo1 自动暂停。
- 滚回 Demo1 从暂停位置继续。
